DESCRIPCIÓN DE LOS AVANCES:
1. Definición del Esqueleto Léxico:
   - Se diseñó la estructura inicial del archivo "main.cpp" dentro de la carpeta "Compilador".
   - Se implementó la lectura básica del archivo fuente "programa.simp" para cargar las instrucciones en memoria de compilación.

2. Creación de la Estructura de Tokens (Analizador Léxico - Fase A):
   - Se estructuró el tipo de datos "Token" utilizando un struct que clasifica el flujo de caracteres bajo dos propiedades esenciales: "tipo" y "valor".
   - Se identificaron los tipos de tokens requeridos para el lenguaje Simple: NUM_LINEA, COMANDO, VARIABLE, CONSTANTE y OPERADOR.

3. Esbozo de la Tabla de Símbolos (Primera Pasada - Fase B):
   - Se conceptualizó la necesidad de una tabla que asocie símbolos (variables, constantes y números de línea) con sus correspondientes direcciones de memoria física de Simpletron.
   - Se estructuró el flujo para separar las áreas de asignación de memoria: las instrucciones se asignarán desde la dirección 000 en adelante, mientras que las variables y constantes se ubicarán desde la dirección 999 de forma descendente para evitar colisiones.

4. Exportación SML:
   - Se programó la escritura automatizada del compilador en un archivo de salida formateado con el sufijo ".sml" listo para ser leído y ejecutado directamente por la máquina virtual Simpletron.
