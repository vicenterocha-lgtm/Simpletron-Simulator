--------------------------------------------------
* Se reorganizó el repositorio para separar los módulos de código.

[2026-07-07] - Proteccion de memoria y entrada de datos
--------------------------------------------------
* Se modularizo la validacion de rangos mediante la funcion 'validarRango'.
* Se agrego un ciclo de validacion en la operacion READ para asegurar que los datos ingresados en ejecucion no desborden la memoria.

[2026-07-08] - Modificacion 1: Carga desde archivo y modularizacion
--------------------------------------------------
* Se reestructuro por completo el diseño de main.cpp dividiendo el flujo en funciones ('mostrarBienvenida', 'cargarPrograma', 'ejecutarPrograma').
* Se implemento el Punto 1 (Objetivo SMART) para permitir la lectura automatica de instrucciones desde un archivo externo llamado 'programa.sml'.
* Se añadio un mecanismo de respaldo interactivo por si el archivo no se encuentra en el directorio raíz.

[2026-07-09] - Modificacion 2: Expansion de memoria a 1000 posiciones
--------------------------------------------------
* Se incremento el tamaño del arreglo de memoria de 100 a 1000 posiciones (0-999) cumpliendo el objetivo SMART.
* Se recalculo el esquema de decodificacion de instrucciones SML para soportar operandos de 3 digitos mediante divisiones y residuos entre 1000.
* Se ajustaron los controles de flujo y el formato visual de la funcion 'dump' para desplegar correctamente los registros y el mapa completo de memoria expandida.

[2026-07-10] - Modificacion 3: Instruccion de Residuo (Módulo)
--------------------------------------------------
* Se integro una nueva instruccion aritmetica al decodificador SML: MODULUS (codigo de operacion 34).
* Se implemento el comportamiento algoritmico para calcular el residuo matemático (A mod B) almacenando el resultado en el acumulador.
* Se diseño e incorporo un manejo controlado de errores para detener la ejecucion del simulador de forma segura en caso de un intento de division entre cero.

[2026-07-10] - Modificacion 4: Instruccion de Exponenciacion (Potencia)
--------------------------------------------------
* Se añadio la operacion EXPONENT (codigo de operacion 35) al conjunto de instrucciones SML del simulador.
* Se implemento el calculo de potencias mediante un bucle de multiplicaciones sucesivas para controlar el desbordamiento de enteros en cada iteracion.
* Se definieron validaciones de seguridad para detener de forma limpia el sistema si se detecta un exponente negativo o un desbordamiento en el acumulador.

[2026-07-10] - Modificacion 5: Instrucciones para manejo de cadenas de caracteres
--------------------------------------------------
* Se crearon e integraron las operaciones READSTRING (codigo de operacion 12) y WRITESTRING (codigo de operacion 13).
* Se diseño el almacenamiento secuencial traduciendo caracteres a valores enteros ASCII dentro del vector de memoria.
* Se implemento un valor nulo (0) como centinela final para delimitar el termino de las cadenas durante los procesos de lectura y escritura.
